# Milestone 26 — Governed Security Operations Foundation

## Baseline

- Branch before milestone: `milestone-24-25-combined-integration`
- Required commit: `442e7b971532ca6f78be2e4b769e5c34addf558b`
- Milestone branch: `milestone-26-governed-security-operations`

## Implemented

- typed, hash-bound action manifests and deterministic policy;
- persistent approval requests, six decisions, persistent expiry, requester
  separation, and one-time consumption;
- conditional approvals that fail closed until machine validation exists;
- per-request hash-chained approval events;
- registry for twelve free security tools;
- fixed shell-free command planning for bounded direct adapters;
- connector-only classification for service and high-risk integrations;
- disabled-by-default bounded subprocess executor;
- structured Nmap XML and JSONL normalisation;
- append-only evidence ledger and artifact verification;
- durable dependency-aware task graph;
- advanced multi-stage assessment planning;
- local-first provider routing and cloud privacy gate contracts;
- owner break-glass and privileged-broker data contracts without sudo secrets;
- specialist roles and skills;
- authenticated Approval Centre, Tool Registry, and Advanced Mode pages.

## Deliberately not activated

- no external tool installation;
- no target scan or network contact;
- no connector activation;
- no privileged service;
- no sudo password storage;
- no arbitrary shell;
- no cloud model call;
- no commit, merge, or push;
- no policy change that permits public or unauthorized scanning.

## Follow-on activation work

Before the first live tool call, a separate reviewed milestone must bind these
contracts to current campaign authorization and scope records, verify installed
tool versions, approve exact profiles, add cancellation and activity events
around execution, and independently validate every parser and connector.
