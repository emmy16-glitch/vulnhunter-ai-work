# Responsive and Accessibility Requirements

The desktop interface is the primary dense operations view, but every essential
workflow must remain usable on tablet and mobile. Smaller layouts may reorganize or
progressively disclose secondary metadata; they must not hide scope warnings,
review conflicts, release blockers, or integrity failures.

Required practices include keyboard navigation, visible focus, semantic landmarks,
labelled controls, accessible status text in addition to color, 44-pixel touch
targets, reduced-motion support, screen-reader announcements for scan progress, and
error summaries that link to invalid fields.

Tables use priority columns and detail drawers on mobile rather than shrinking text
to unreadable sizes.
